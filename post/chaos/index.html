Chaos

Jan 1, 0001 2 min read

Motivation

Embedded systems are increasingly required to provide both complicated feature-sets, and high-confidence in the correctness of mission-critical computations. Multi-core systems complicate the isolation between subsystems of different assurance levels (degree of confidence that a code meets its specification). The kernel data-structures shared between cores like scheduler run-queues, etc, and the Inter-Processor Interrupts (IPIs) used for message passing between cores, can induce unbounded interference on the high-assurance, high-criticality tasks.

Research

This research introduces Chaos, which is designed to devirtualize high-criticality tasks, a process that exports them out of possibly low-assurance subsystems that host low-assurance tasks, to remove interference due to the shared subsystem. Such tasks are imported into a ChaosRT execution environment with minimal controlled interference and strictly predictable execution, while leveraging the high-functionality of low-assurance code via proxies. The proxies implement IPI rate-limiting servers where appropriate to bound the cross-core interference.

Contributions

  • The design and implementation of devirtualization and ChaosRT to extract high-criticality subsystems from lower-assurance legacy systems – while maintaining functional dependencies via proxies – and predictable inter-core message passing mechanisms using IPI rate-limiting servers.
  • The NASA’s core Flight System (cFS) implementation in Chaos devirtualizing the OpenSatKit and the high-criticality safety controller applications from the cFS middleware while leveraging the Rumpkernel unikernel (NetBSD drivers) for communication with the ground station via proxies.
  • Evaluation of Chaos and different reliability focused systems like Composite, Fiasco.OC, seL4 and Linux with real-time patch to study the predictability and bounded interference properties, and evaluation of cFS on Chaos and Linux with real-time patch to show that Chaos lowers the sensor processing latency by a factor of 2.7 and 3.5 for average and worst-case respectively compared to Linux.

Technical Skills

  • Built the minimal ChaosRT runtime for predictable execution of high-criticality tasks and proxies for inter-assurance coordination leveraging synchronous or asynchronous coordination techniques in Composite kernel.
  • Built the IPI rate-limiting servers for controlled interference from low-assurance software on the high-criticality tasks and the polling mechanism to process the requests for dropped IPIs.
  • Implemented NASA’s core Flight System (cFS) in Chaos by devirtualizing different cFS, OpenSatKit applications and the safety controller application and enabled proxies between them and cFS or NetBSD drivers for leveraging the functionality from these subsystems.
